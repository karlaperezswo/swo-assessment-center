Implementación de MGN Connector (On-Premises → AWS)

Guía completa para instalar y configurar el MGN Connector que permite la conexión entre el entorno on-premises y AWS Application Migration Service.

1. Requisitos Generales

  • Servidor dedicado recomendado
  • Permisos IAM configurados
  • Conectividad saliente HTTPS (puerto 443)
Documentación oficial: Application Migration Service Connectors

2. Sistemas Operativos Soportados

Linux (64 bits):

  • Ubuntu 18.x – 22.04
  • Amazon Linux 2
  • RHEL 8.x

No soportados:

  • CentOS 5 / 6
  • RHEL 6
  • Oracle 6
  • Amazon Linux 1

3. Requisitos del Agente SSM

  • Instalación automática del agente SSM
  • Si ya existe una versión previa → desinstalar primero
  • Espacio mínimo requerido:
    • 200 MB en disco
    • 200 KB en /var

4. Creación de Roles IAM con CloudFormation

4.1 Template: Roles principales

AWSTemplateFormatVersion: "2010-09-09"
Description: IAM Roles for AWS Application Migration Connector

Resources:

  MGNConnectorInstallerRole:
    Type: AWS::IAM::Role
    Properties:
      RoleName: MGNConnectorInstallerRole
      AssumeRolePolicyDocument:
        Version: "2012-10-17"
        Statement:
          - Effect: Allow
            Principal:
              AWS: arn:aws:iam::<ACCOUNT_ID>/mgn-conector
            Action: sts:AssumeRole
      Policies:
        - PolicyName: MGNConnectorInstallerPolicy
          PolicyDocument:
            Version: "2012-10-17"
            Statement:
              - Effect: Allow
                Action: mgn:*
                Resource: "*"

  AWSApplicationMigrationConnectorManagementRole:
    Type: AWS::IAM::Role
    Properties:
      RoleName: AWSApplicationMigrationConnectorManagementRole
      AssumeRolePolicyDocument:
        Version: "2012-10-17"
        Statement:
          - Effect: Allow
            Principal:
              Service: ssm.amazonaws.com
            Action: sts:AssumeRole
      ManagedPolicyArns:
        - arn:aws:iam::aws:policy/AmazonSSMManagedInstanceCore
      Policies:
        - PolicyName: MgnConnectorPolicy
          PolicyDocument:
            Version: "2012-10-17"
            Statement:
              - Effect: Allow
                Action: logs:*
                Resource: "*"
              - Effect: Allow
                Action: s3:GetObject
                Resource:
                  - arn:aws:s3:::aws-application-migration-service-us-east-1/*
              - Effect: Allow
                Action: s3:PutObject
                Resource:
                  - arn:aws:s3:::log-mgn-conector/*
              - Effect: Allow
                Action: sts:AssumeRole
                Resource:
                  - arn:aws:iam::<ACCOUNT_ID>:role/AWSApplicationMigrationConnectorSharingRole_<ACCOUNT_ID>
              - Effect: Allow
                Action: secretsmanager:GetSecretValue
                Resource: "*"

4.2 Template: Role para agentes

AWSTemplateFormatVersion: '2010-09-09'
Description: IAM Role for AWS Application Migration Connector

Resources:

  ApplicationMigrationConnectorRole:
    Type: AWS::IAM::Role
    Properties:
      RoleName: AWSApplicationMigrationConnectorSharingRole_<ACCOUNT_ID>
      AssumeRolePolicyDocument:
        Version: '2012-10-17'
        Statement:
          - Effect: Allow
            Principal:
              Service:
                - ec2.amazonaws.com
                - ssm.amazonaws.com
            Action:
              - sts:AssumeRole
      ManagedPolicyArns:
        - arn:aws:iam::aws:policy/AWSApplicationMigrationAgentInstallationPolicy

Outputs:
  RoleName:
    Value: !Ref ApplicationMigrationConnectorRole
  RoleArn:
    Value: !GetAtt ApplicationMigrationConnectorRole.Arn

4.3 Despliegue

aws cloudformation create-stack \
  --stack-name mgn-connector-roles \
  --template-body file://roles.yaml \
  --capabilities CAPABILITY_NAMED_IAM

5. Hybrid Activation (SSM)

Desde AWS Systems Manager, generar:

  • Activation ID
  • Activation Code
Una vez generados el Activation Code y el Activation ID, ir a la sección de MGN Connector, hacer clic en Agregar conector e ingresar los datos solicitados.

6. Arquitectura

Flujo del conector:

  1. On-prem Connector
  2. Registro vía SSM
  3. Conexión a AWS
  4. Conexión a vCenter
  5. Descubrimiento de VMs
  6. Instalación de agentes

7. Instalación del MGN Connector

7.1 Descargar el instalador

wget -O ./aws-vcenter-client-installer-init.py \
  https://aws-application-migration-service-us-east-1.s3.us-east-1.amazonaws.com/latest/vcenter-client/linux/aws-vcenter-client-installer-init.py

7.2 Ejecutar la instalación

sudo python3 aws-vcenter-client-installer-init.py \
  --connector source-automation \
  --region us-east-1 \
  --activation-code <ACTIVATION_CODE> \
  --connector-name <CONNECTOR_NAME>
Nota: Reemplazar <ACTIVATION_CODE> y <CONNECTOR_NAME> con los valores generados en el paso de Hybrid Activation.

8. Instalación del Agente MGN

Se recomienda usar la opción Install agent (automático) desde la consola de AWS MGN para simplificar el proceso.

Resumen del Flujo Completo

Listo: Con el MGN Connector configurado, puedes continuar con la ejecución del plan de migración en Proceso de uso de Transform.
