Manual de Instalación del OVA en vCenter

Este documento describe el procedimiento para la instalación, configuración y puesta en marcha del Discovery Tool de AWS Transform en un entorno VMware vCenter, incluyendo los prerrequisitos, despliegue del OVA y configuración inicial.

1. Prerrequisitos

1.1 Requisitos Generales

Antes de instalar el Discovery Tool, se deben cumplir los siguientes requisitos:

  • Máquina virtual con:
    • 4 vCPU
    • 16 GB de RAM
    • 35 GB de almacenamiento
  • Conectividad de red habilitada
  • Acceso a los servidores objetivo desde la VM del Discovery Tool

1.2 Puertos de Red Requeridos

El Discovery Tool recopila información mediante conexiones entrantes a los servidores:

Sistema Operativo Protocolo Puerto Uso
Linux SSH TCP 22 Recolección de datos
Windows (HTTP) TCP 5985 WinRM
Windows (HTTPS) TCP 5986 WinRM
SNMP UDP 161 Recolección de red

1.3 Requisitos para Linux

  • Usuario con acceso SSH
  • Permisos para ejecutar comandos:
    • ss o netstat
    • dmidecode
    • lvdisplay
  • Se recomienda configurar sudo sin contraseña (passwordless sudo) para asegurar la recolección completa de datos

1.4 Requisitos para VMware

  • Versión de VMware vCenter: 6.5, 6.7, 7.0 u 8.0
  • Permisos:
    • Read y View en el grupo System
    • Permiso para desplegar OVA

2. Descarga del Discovery Tool

  1. Acceder al entorno de vCenter
  2. Descargar el archivo OVA desde el siguiente enlace:
URL de descarga:
https://s3.us-east-1.amazonaws.com/atx.discovery.collector.bundle/releases/latest/AWS-Transform-discovery-tool.ova

3. Despliegue del OVA

  1. Iniciar sesión en vCenter
  2. Seleccionar el clúster o recurso destino
  3. Ejecutar el proceso estándar: Deploy OVF Template
  4. Seleccionar el archivo .ova descargado
  5. Configurar:
    • Nombre de la VM
    • Red
    • Almacenamiento
  6. Finalizar el despliegue

4. Configuración Inicial

4.1 Acceso a la Interfaz

Parámetro Valor
URL de acceso https://<IP_VM>:5000
Usuario discovery
Contraseña inicial Password

4.2 Cambio de Contraseña

Importante: Crear una nueva contraseña al ingresar por primera vez.

5. Configuración de Acceso a vCenter

  1. Ingresar a la opción: Set up access to connect to your vCenter Server
  2. Introducir:
    • FQDN o IP del vCenter (sin https://)
    • Usuario y contraseña
Formato de usuario Windows: DOMINIO\usuario
  1. Verificar conexión — Estado esperado: Connected. Se mostrará la última fecha de recolección.
  2. Para ejecutar recolección manual: Actions → Force collection

6. Inventario Descubierto

Acceder a Discovered Inventory para visualizar:

  • Máquinas virtuales
  • Servicios
  • Dependencias

7. Configuración de Acceso a Sistemas Operativos

7.1 Tipos de Credenciales

  • SSH (Linux)
  • WinRM (Windows)
Formato Kerberos: usuario@DOMINIO.TLD

7.2 Configuración Automática

Activar Auto-connect para aplicar credenciales a todas las VMs automáticamente.

7.3 Configuración Manual

  1. Ir a Discovered Inventory
  2. Aplicar filtro (ejemplo: atx-wp)
  3. Seleccionar hosts
  4. Elegir Manage access credential
  5. Asignar credenciales
  6. Guardar y ejecutar recolección

7.4 Validación

Estado esperado:

  • Network Status → Success
  • Database Status → Success (para Windows con SQL Server)

8. Descargas

  • Inventario: opción Download Inventory
  • Logs: opción Download Logs

9. Resultados Esperados

Identificación de:

  • Servidores
  • Aplicaciones
  • Dependencias

Ejemplo de datos recolectados:

  • IP origen y destino
  • Puertos
  • Protocolos
  • Servicios activos

10. Recomendaciones

  • Verificar conectividad de red antes del despliegue
  • Utilizar cuentas con permisos adecuados
  • Configurar acceso automático cuando sea posible
  • Revisar logs en caso de errores
Siguiente paso: Una vez instalado el Discovery Tool, continúa con el Proceso de uso de Transform.
