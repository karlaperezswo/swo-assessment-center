Proceso de Migración VMware a AWS con AWS Transform

Este documento describe el proceso de migración de servidores VMware hacia Amazon Web Services (AWS) utilizando AWS Transform. La migración se realiza mediante una estrategia de rehost (lift-and-shift) utilizando AWS Application Migration Service (MGN) en modalidad single-account.

Fase 1: Descubrimiento e Importación de Inventario

Objetivo: Identificar y recopilar información sobre la infraestructura VMware de origen.

Actividades

  • Importar los archivos de inventario de la infraestructura VMware de origen:
    • Exportación de RVTools con datos del entorno VMware (.xlsx)
    • Exportación del Discovery Tool (.zip)
  • AWS Transform procesa los datos y extrae:
    • Información de servidores y configuraciones
    • Sistemas operativos
    • Recursos de cómputo (CPU, memoria)
    • Almacenamiento y relaciones de red
Resultado: Inventario completo de servidores VMware procesado y disponible para planificación.

Fase 2: Planificación de Migración

Objetivo: Agrupar servidores en aplicaciones y definir oleadas (waves) de migración.

Actividades

  • Analizar las dependencias entre servidores y aplicaciones
  • Agrupar los servidores en aplicaciones lógicas basadas en dependencias técnicas y de negocio
  • Generar un plan de oleadas (wave plan) de migración

Artefacto generado

  • migration_planning/wave_plan-[fecha].csv — Plan de oleadas con la asignación de servidores a waves
Resultado: Se define la oleada Wave 0 con los grupos de servidores a migrar.

Fase 3: Conexión a la Cuenta AWS de Destino

Objetivo: Establecer la conexión segura entre AWS Transform y la cuenta AWS de destino.

Actividades

  • Configurar el conector de cuenta de destino (Target account connection)
  • Establecer la conexión con la cuenta AWS de destino
  • Configurar el rol IAM correspondiente

Parámetros de configuración

Parámetro Descripción
Cuenta AWS de destino ID de la cuenta AWS receptora
Región de destino Región AWS seleccionada (ej. us-east-2)
Tipo de migración Single-account
Rol IAM AWSTransform-Connector-role
Resultado: Conexión activa y verificada con la cuenta de destino.

Fase 4: Migración de Red

Objetivo: Configurar la infraestructura de red necesaria en AWS para recibir los servidores migrados.

Actividades

  • Diseñar y desplegar la topología de red en AWS
  • Configurar VPCs, subnets, security groups y demás componentes de red
  • Asegurar la conectividad entre el entorno de origen y la infraestructura AWS de destino
Resultado: Infraestructura de red desplegada y lista para recibir los servidores migrados.

Fase 5: Preparación y Ejecución del Plan de Migración

Objetivo: Preparar los artefactos necesarios para la ejecución de la migración con AWS MGN.

Actividades

  • Generar el inventario enriquecido de MGN (enriched_inventory-[id]-[fecha].csv)
  • Generar recomendaciones de instancias EC2 para los servidores
  • Crear el archivo de importación MGN específico para cada Wave (mgn_import_Wave_[nombre]-[fecha].csv)
  • Verificar que todos los servidores sean emparejados exitosamente con tipos de instancia EC2
Resultado: Todos los servidores de la Wave configurados con tipos de instancia EC2 apropiados.

Fase 6: Ejecución de la Migración

Objetivo: Ejecutar la migración de los servidores de cada Wave.

Actividades

  1. Iniciar la replicación de datos de los servidores mediante AWS MGN
  2. Ejecutar pruebas de lanzamiento (test launch) para validar la migración
  3. Revisar las instancias de prueba generadas
  4. Aprobar las acciones de cutover a través de la consola de aprobaciones
  5. Ejecutar el cutover de las instancias
Nota: Verificar el estado de cada servidor antes de aprobar el cutover. El proceso es irreversible una vez completado.

Resumen de Artefactos Generados

Artefacto Descripción
RVTools export (.xlsx) Archivo de inventario VMware de origen
Discovery Tool export (.zip) Datos de descubrimiento de 28 días
wave_plan-[fecha].csv Plan de oleadas de migración
enriched_inventory-[id]-[fecha].csv Inventario enriquecido de MGN
mgn_import_Wave_[nombre]-[fecha].csv Archivo de importación MGN por Wave
Siguiente paso: Para configurar el conector MGN en el entorno on-premises, consulta la guía de Uso de MGN Connector.
