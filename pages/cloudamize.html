Cloudamize

Cloudamize es una plataforma de análisis y optimización de infraestructura que proporciona recomendaciones precisas para migración a AWS basadas en datos reales de uso y performance.

Prerrequisitos

Requisitos Técnicos

  • Sistemas Operativos Soportados:
    • Windows Server 2008 R2 y superior
    • Linux (RHEL, CentOS, Ubuntu, SUSE, Debian)
    • VMware vSphere 5.5 y superior
  • Recursos del Sistema:
    • CPU: Mínimo 1% de uso disponible
    • RAM: 50 MB disponibles
    • Disco: 100 MB de espacio libre
  • Conectividad:
    • Acceso HTTPS saliente al puerto 443
    • URL permitida: *.cloudamize.com
    • Ancho de banda: ~1 KB/s por servidor

Permisos Necesarios

  • Windows: Administrador local o permisos de lectura WMI
  • Linux: Usuario root o sudo para instalación
  • VMware: Rol de solo lectura en vCenter

Troubleshooting

Problema: Agente no se conecta a Cloudamize

Síntomas: El agente está instalado pero no aparece en el portal

Soluciones:

  1. Verificar conectividad: telnet cloudamize.com 443
  2. Revisar configuración de proxy en el agente
  3. Verificar que el firewall permita tráfico saliente HTTPS
  4. Revisar logs del agente en: /var/log/cloudamize/ (Linux) o C:\ProgramData\Cloudamize\logs\ (Windows)
  5. Reiniciar el servicio del agente

Problema: Datos incompletos o inconsistentes

Síntomas: Faltan métricas o los datos no son coherentes

Soluciones:

  1. Verificar que el período de recolección sea de al menos 2 semanas
  2. Confirmar que los agentes han estado activos todo el período
  3. Revisar si hay servidores apagados o en mantenimiento
  4. Validar permisos del agente para acceder a métricas del sistema

Problema: Alto consumo de recursos por el agente

Síntomas: El agente consume más CPU/RAM de lo esperado

Soluciones:

  1. Ajustar la frecuencia de recolección de métricas
  2. Verificar que no haya múltiples instancias del agente corriendo
  3. Actualizar a la última versión del agente
  4. Contactar soporte de Cloudamize para optimización
