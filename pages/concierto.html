Concierto

Concierto es una plataforma avanzada de análisis multi-cloud que utiliza machine learning para proporcionar recomendaciones inteligentes de migración y optimización. Es ideal para entornos complejos y heterogéneos.

Prerrequisitos

Requisitos Técnicos

  • Sistemas Operativos Soportados:
    • Windows Server 2012 y superior
    • Linux (todas las distribuciones principales)
    • AIX, Solaris, HP-UX
    • VMware, Hyper-V, KVM
  • Modos de Operación:
    • Agentless: Conexión remota vía SSH/WMI
    • Agent-based: Instalación de agente ligero
    • Hybrid: Combinación de ambos métodos
  • Recursos del Sistema (modo agente):
    • CPU: <1% de uso
    • RAM: 30 MB disponibles
    • Disco: 50 MB de espacio libre
  • Conectividad:
    • Modo agentless: SSH (puerto 22) o WMI (puerto 135, 445)
    • Modo agente: HTTPS saliente al puerto 443
    • URL permitida: *.concierto.cloud

Permisos Necesarios

  • Windows: Administrador local o permisos WMI de lectura
  • Linux: Usuario con sudo o acceso root (solo lectura)
  • VMware: Rol de solo lectura en vCenter
  • Cloud Providers: Credenciales con permisos de lectura (ReadOnly)

Troubleshooting

Problema: Fallo en conexión agentless

Síntomas: No se puede establecer conexión SSH/WMI con los servidores

Soluciones:

  1. Verificar credenciales y permisos de usuario
  2. Confirmar que los puertos necesarios estén abiertos en firewall
  3. Validar que SSH/WMI esté habilitado en los servidores target
  4. Probar conexión manual desde el servidor Concierto
  5. Revisar logs de conexión en la plataforma

Problema: Discovery incompleto

Síntomas: Faltan servidores o aplicaciones en el inventario

Soluciones:

  1. Verificar que todos los rangos de IP estén incluidos en el scan
  2. Confirmar que las credenciales tengan acceso a todos los segmentos
  3. Revisar si hay servidores en subredes aisladas
  4. Ejecutar discovery manual para servidores faltantes
  5. Validar que no haya limitaciones de firewall entre segmentos

Problema: Análisis de dependencias impreciso

Síntomas: Las dependencias mostradas no coinciden con la realidad

Soluciones:

  1. Extender el período de observación (mínimo 3 semanas recomendado)
  2. Validar que el análisis de red capture todo el tráfico
  3. Complementar con información manual del cliente
  4. Revisar configuración de filtros de tráfico
  5. Ejecutar análisis en horarios de mayor actividad
