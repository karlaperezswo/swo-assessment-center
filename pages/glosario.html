Glosario de Términos

Definiciones de términos técnicos y acrónimos utilizados en el contexto de MAP Assessment y migración a AWS.

A

Agentless: Método de recolección de datos que no requiere instalación de software en los servidores objetivo. Utiliza conexiones remotas (SSH, WMI) para obtener información.

ALB (Application Load Balancer): Balanceador de carga de AWS que opera en la capa 7 (aplicación) del modelo OSI, ideal para tráfico HTTP/HTTPS.

AMI (Amazon Machine Image): Plantilla que contiene la configuración de software necesaria para lanzar una instancia EC2.

As-Is: Estado actual de la infraestructura antes de la migración.

Assessment: Fase inicial del programa MAP donde se evalúa el entorno actual del cliente y se desarrolla el plan de migración.

Auto Scaling: Capacidad de AWS para ajustar automáticamente la cantidad de recursos computacionales según la demanda.

Availability Zone (AZ): Ubicación física aislada dentro de una región de AWS, diseñada para proporcionar alta disponibilidad.

B

Backup: Copia de seguridad de datos que permite la recuperación en caso de pérdida o corrupción.

Business Case: Documento que justifica la inversión en la migración, incluyendo análisis de costos, beneficios y ROI.

C

CapEx (Capital Expenditure): Gastos de capital en activos físicos como servidores y equipamiento de datacenter.

CloudFormation: Servicio de AWS para crear y gestionar recursos mediante plantillas de infraestructura como código.

CloudTrail: Servicio de AWS que registra todas las llamadas a la API para auditoría y compliance.

CloudWatch: Servicio de monitoreo y observabilidad de AWS que recopila métricas, logs y eventos.

Compliance: Cumplimiento de regulaciones, estándares y políticas de seguridad aplicables.

D

Datacenter: Instalación física que aloja servidores, almacenamiento y equipamiento de red.

Dependency Mapping: Proceso de identificar y documentar las relaciones entre aplicaciones y componentes de infraestructura.

Direct Connect: Conexión de red dedicada entre el datacenter del cliente y AWS.

Discovery: Proceso de identificación y catalogación de todos los activos de TI en el entorno del cliente.

DMS (Database Migration Service): Servicio de AWS para migrar bases de datos a AWS de manera segura y con mínimo downtime.

DR (Disaster Recovery): Conjunto de políticas y procedimientos para recuperar sistemas críticos después de un desastre.

E

EBS (Elastic Block Store): Servicio de almacenamiento en bloque persistente para instancias EC2.

EC2 (Elastic Compute Cloud): Servicio de computación en la nube de AWS que proporciona capacidad de cómputo escalable.

ECS (Elastic Container Service): Servicio de orquestación de contenedores de AWS.

EKS (Elastic Kubernetes Service): Servicio administrado de Kubernetes en AWS.

G

GDPR (General Data Protection Regulation): Regulación europea de protección de datos personales.

GuardDuty: Servicio de detección de amenazas de AWS que monitorea actividad maliciosa.

H

HIPAA (Health Insurance Portability and Accountability Act): Ley estadounidense que regula la protección de información médica.

Hybrid Cloud: Arquitectura que combina infraestructura on-premises con servicios cloud.

I

IAM (Identity and Access Management): Servicio de AWS para gestionar usuarios, grupos y permisos de acceso.

IaaS (Infrastructure as a Service): Modelo de servicio cloud que proporciona infraestructura virtualizada.

Immersion Day: Sesión de capacitación práctica sobre servicios y mejores prácticas de AWS.

K

Kickoff: Reunión inicial de un proyecto para alinear objetivos, alcance y expectativas.

KMS (Key Management Service): Servicio de AWS para crear y controlar claves de cifrado.

L

Lambda: Servicio de computación serverless de AWS que ejecuta código en respuesta a eventos.

Landing Zone: Entorno AWS multi-cuenta configurado según mejores prácticas, listo para recibir cargas de trabajo.

Lift and Shift: Estrategia de migración que mueve aplicaciones a la nube sin modificaciones (Rehost).

M

MAP (Migration Acceleration Program): Programa de AWS que proporciona metodología, herramientas y financiamiento para acelerar migraciones.

MGN (Application Migration Service): Servicio de AWS para migración lift-and-shift automatizada.

Migration Wave: Grupo de aplicaciones que se migran juntas en una fase específica del proyecto.

Mobilize: Segunda fase del programa MAP donde se prepara la organización para la migración.

N

NACLs (Network Access Control Lists): Capa de seguridad que actúa como firewall a nivel de subnet en AWS.

NLB (Network Load Balancer): Balanceador de carga de AWS que opera en la capa 4 (transporte) del modelo OSI.

O

On-premises: Infraestructura de TI ubicada y operada en las instalaciones físicas de la organización.

OpEx (Operational Expenditure): Gastos operativos recurrentes como servicios cloud y suscripciones.

P

PCI DSS (Payment Card Industry Data Security Standard): Estándar de seguridad para organizaciones que manejan información de tarjetas de crédito.

Q

Quick Win: Migración temprana de bajo riesgo que genera valor rápido y construye confianza en el proceso.

R

RACI Matrix: Matriz que define roles y responsabilidades (Responsible, Accountable, Consulted, Informed).

RDS (Relational Database Service): Servicio administrado de bases de datos relacionales de AWS.

Refactor: Estrategia de migración que implica rediseñar la aplicación para aprovechar capacidades cloud-native.

Region: Área geográfica donde AWS tiene múltiples Availability Zones.

Rehost: Estrategia de migración lift-and-shift sin modificaciones a la aplicación.

Relocate: Estrategia de migración que mueve máquinas virtuales a AWS sin cambios (VMware Cloud on AWS).

Replatform: Estrategia de migración con optimizaciones menores para aprovechar algunos beneficios cloud.

Repurchase: Estrategia de migración que reemplaza la aplicación actual con una solución SaaS.

Retain: Estrategia de mantener aplicaciones on-premises por el momento.

Retire: Estrategia de desactivar aplicaciones que ya no se necesitan.

Rightsizing: Proceso de ajustar el tamaño de recursos para optimizar costos y performance.

ROI (Return on Investment): Métrica que mide el retorno financiero de una inversión.

S

S3 (Simple Storage Service): Servicio de almacenamiento de objetos de AWS.

SaaS (Software as a Service): Modelo de servicio donde el software se entrega a través de internet.

Security Group: Firewall virtual que controla el tráfico de entrada y salida de instancias EC2.

Serverless: Modelo de computación donde el proveedor cloud gestiona la infraestructura automáticamente.

SLA (Service Level Agreement): Acuerdo que define el nivel de servicio esperado.

SOC (Service Organization Control): Estándar de auditoría para controles de seguridad y disponibilidad.

T

TCO (Total Cost of Ownership): Costo total de poseer y operar un sistema, incluyendo todos los gastos directos e indirectos.

To-Be: Estado futuro deseado de la infraestructura después de la migración.

V

VPC (Virtual Private Cloud): Red virtual aislada en AWS donde se lanzan recursos.

VPN (Virtual Private Network): Conexión segura y cifrada entre redes a través de internet.

W

WAF (Web Application Firewall): Firewall de aplicaciones web de AWS que protege contra ataques comunes.

Wave: Ver Migration Wave.

Well-Architected Framework: Marco de mejores prácticas de AWS para diseñar y operar sistemas en la nube.

7Rs

7Rs: Las siete estrategias de migración a la nube:

  • Retire (Retirar)
  • Retain (Retener)
  • Rehost (Realojar)
  • Relocate (Reubicar)
  • Repurchase (Recomprar)
  • Replatform (Replataformar)
  • Refactor/Re-architect (Refactorizar/Rearquitectar)
Nota: Este glosario se actualiza regularmente. Si encuentras un término que no está definido, por favor contacta al equipo de documentación.
