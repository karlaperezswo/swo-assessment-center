Introducción al Programa MAP

¿Qué es el AWS Migration Acceleration Program (MAP)?

El AWS Migration Acceleration Program (MAP) es un programa integral diseñado por Amazon Web Services para ayudar a las organizaciones a migrar sus cargas de trabajo a la nube de AWS de manera más rápida, eficiente y con menor riesgo.

Nota: MAP proporciona metodología comprobada, herramientas especializadas y financiamiento para acelerar el proceso de migración.

Visión General del Programa

El programa MAP se estructura en tres fases principales:

  • Assess (Evaluación): Análisis del estado actual de la infraestructura y planificación de la migración
  • Mobilize (Movilización): Preparación de la organización y el entorno para la migración
  • Migrate & Modernize (Migración y Modernización): Ejecución de la migración y optimización de las cargas de trabajo

Beneficios del Programa MAP

  • Reducción de costos operativos hasta en un 30%
  • Mejora en la productividad del equipo de TI
  • Aceleración del tiempo de migración
  • Acceso a financiamiento y créditos de AWS
  • Metodología probada y mejores prácticas
  • Herramientas especializadas de evaluación y migración
  • Soporte de expertos certificados

Objetivos del Programa

Los principales objetivos que busca alcanzar el programa MAP son:

  1. Evaluar de manera integral el portafolio de aplicaciones e infraestructura actual
  2. Desarrollar un business case sólido para la migración a AWS
  3. Crear un plan de migración detallado y priorizado
  4. Identificar las estrategias de migración óptimas (7Rs)
  5. Preparar a la organización para el cambio
  6. Ejecutar la migración de manera controlada y eficiente
  7. Optimizar y modernizar las cargas de trabajo en AWS

Fases del Programa MAP

1. Fase de Assess (Evaluación)

La Fase de Assess tiene como objetivo crear un caso de negocio para la migración a la nube y desarrollar un plan de migración de alto nivel.

Actividades Clave:

  • Discovery and Analysis: Recopilar datos sobre las cargas de trabajo existentes y evaluar su preparación para la migración.
  • TCO Analysis: Realizar un análisis de Costo Total de Propiedad para comprender los posibles ahorros de costos.
  • Migration Readiness Assessment (MRA): Evaluar la preparación de la organización en seis perspectivas: negocio, personas, procesos, plataforma, operaciones y seguridad.

Deliverables:

  • Business Case: Un caso de negocio completo que describa los beneficios de la migración.
  • Migration Plan: Un plan de migración de alto nivel con cronogramas y requisitos de recursos.

2. Fase de Mobilize (Movilización)

Objetivos:

La Fase de Movilización se centra en la planificación detallada, la construcción de capacidades fundamentales y en abordar las brechas identificadas durante la Fase de Evaluación.

Actividades clave:

  • Migración piloto: Realizar una migración piloto para validar el plan y abordar cualquier desafío.
  • Planeación detallada: Desarrollar planes de migración detallados, que incluyan la asignación de recursos, cronogramas y gestión de riesgos.
  • Foundation Building: Establecer una base lista para la migración con modelos adecuados de seguridad, gobernanza y operaciones.

Entregables:

  • Migration Runbook: Un runbook detallado que describa el proceso de migración paso a paso.
  • Base operacional: Procedimientos operativos y herramientas establecidos para las operaciones en la nube.
  • Training: Programas de capacitación para los equipos, asegurando que estén preparados para gestionar el entorno en la nube.

3. Fase de Migrate & Modernize (Migración y Modernización)

Objetivos:

La Fase de Migración y Modernización implica ejecutar el plan de migración, mover las cargas de trabajo a AWS y optimizarlas para mejorar el rendimiento y reducir costos.

Actividades clave:

  • Migración de cargas de trabajo: Ejecutar la migración de las cargas de trabajo según el plan detallado.
  • Modernización: Optimizar y modernizar aplicaciones para aprovechar los servicios y características de AWS.
  • Post-Migración Support: Proporcionar soporte y orientación para garantizar un funcionamiento fluido en el entorno de AWS.

Entregables:

  • Cargas de trabajo migradas: Cargas de trabajo migradas con éxito a AWS.
  • Aplicaciones optimizadas: Aplicaciones modernizadas y optimizadas para el rendimiento en la nube y la eficiencia de costos.
  • Excelencia operativa: Mejores prácticas establecidas para operaciones en la nube y mejora continua.
Importante: Esta wiki se enfoca específicamente en la fase de Assessment, proporcionando todas las herramientas y guías necesarias para ejecutarla de manera estandarizada.
