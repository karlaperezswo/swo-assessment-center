Recolección de Datos

El período de recolección de datos es fundamental para obtener información precisa y representativa del entorno del cliente. Durante esta fase, los agentes recopilan métricas de uso, performance y dependencias.

Objetivo: Recolectar datos completos y precisos durante un período suficiente para capturar patrones de uso representativos.

Duración del Período de Recolección

Escenario Duración Mínima Duración Recomendada
Entorno estándar 2 semanas 3-4 semanas
Cargas variables 3 semanas 4-6 semanas
Patrones mensuales 4 semanas 6-8 semanas
Entorno complejo 4 semanas 8-12 semanas

Datos Recolectados

Métricas de Infraestructura

  • CPU: Utilización, cores, velocidad
  • Memoria: Uso, capacidad, disponible
  • Disco: IOPS, throughput, capacidad, uso
  • Red: Ancho de banda, latencia, conexiones
  • Sistema Operativo: Versión, patches, configuración

Información de Aplicaciones

  • Procesos en ejecución
  • Servicios instalados
  • Puertos en uso
  • Conexiones de red
  • Dependencias entre aplicaciones

Datos de Performance

  • Patrones de uso por hora/día/semana
  • Picos de carga
  • Períodos de baja utilización
  • Tendencias de crecimiento

Problemas Comunes y Soluciones

Agentes Desconectados

Causas:

  • Servidor reiniciado
  • Problemas de red
  • Servicio detenido
  • Cambios en firewall

Solución: Reiniciar agente, verificar conectividad, revisar logs

Datos Incompletos

Causas:

  • Período de recolección insuficiente
  • Servidores apagados
  • Permisos insuficientes
  • Configuración incorrecta

Solución: Extender período, verificar permisos, ajustar configuración

Datos Inconsistentes

Causas:

  • Mantenimientos durante recolección
  • Cambios en la infraestructura
  • Eventos excepcionales

Solución: Documentar eventos, considerar extender período
