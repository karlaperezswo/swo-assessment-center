Rapid Discovery

¿Qué hace esta aplicación?

Pre-Discovery & Segmentación de Datos es una herramienta profesional que le permite procesar y enriquecer la información de inventarios de servidores virtuales exportados desde RVTools, facilitando la documentación completa para proyectos de assessment, migración o auditoría.

Características Principales

  • Procesamiento Automático: Lee archivos Excel de RVTools y extrae la información relevante
  • Validaciones Inteligentes: Detecta automáticamente SOs no soportados
  • Búsqueda Avanzada: Filtra y encuentra servidores rápidamente
  • Sin Base de Datos: Todo se guarda en archivos locales

Requisitos Previos

Antes de comenzar, asegúrese de tener:

Pasos para Usar la Aplicación

Paso 1: Abrir la Aplicación

Haga doble clic en el archivo index.html o ábralo con su navegador preferido.

Paso 2: Cargar Archivo RVTools

Haga clic en "Seleccionar Archivo" y elija su archivo .xlsx de RVTools. La aplicación procesará automáticamente la hoja "vInfo".

Paso 3: Revisar Validaciones Automáticas

La aplicación marcará automáticamente:

  • Sistemas operativos de VMware (ESXi, vCenter, etc.)
  • Sistemas operativos de escritorio (Windows 7, 10, 11, etc.)
  • Máquinas virtuales apagadas

Paso 4: Completar Información

Para cada servidor, complete los siguientes campos:

  • Hostname: Nombre real del servidor en la red
  • SO Corregido: Versión completa del sistema operativo
  • Ambiente: Producción, Desarrollo, Testing/QA, Staging
  • Aplicación: Nombre de la aplicación principal
  • Lenguaje: Lenguaje de programación (Java, Python, .NET, etc.)
  • Base de Datos: Marque si tiene BD y especifique el motor

Paso 5: Guardar Progreso

Use el botón "Guardar Progreso" para descargar un archivo JSON con su trabajo. Puede cargarlo después con "Cargar Progreso".

Paso 6: Exportar Resultados

Una vez completada la información:

  • Exportar Excel: Genera un archivo .xlsx completo
  • Exportar PDF: Genera un reporte profesional

Códigos de Color

Fondo Amarillo: Sistema operativo no soportado (VMware o escritorio). Estos servidores quedan automáticamente fuera del alcance del assessment.
Fondo Rojo Claro: Máquina virtual apagada. Puede decidir si incluirla o no en el assessment usando el checkbox "Incluir".

Función de Búsqueda

Use la barra de búsqueda para filtrar servidores por:

  • Nombre de VM
  • Hostname
  • Sistema operativo
  • Cluster
  • Datacenter
  • Ambiente
  • Nombre de aplicación
Tip: Trabaje por grupos usando la búsqueda. Por ejemplo, busque "prod" para ver solo servidores de producción, o "sql" para ver servidores con SQL Server.

Panel de Estadísticas

El panel superior muestra en tiempo real:

  • Total VMs: Cantidad total de máquinas virtuales
  • Encendidas: VMs en estado "Powered On"
  • Apagadas: VMs en estado "Powered Off"
  • No Soportadas: VMs con SO no soportado
  • Completadas: VMs con información completa (Hostname, SO y Ambiente)

Almacenamiento de Datos

¿Dónde se guardan mis datos?

La aplicación NO usa base de datos. Sus datos se guardan en:

  • Archivos JSON: Para guardar y cargar progreso
  • Archivos Excel: Para exportar resultados completos
  • Archivos PDF: Para reportes ejecutivos
Importante: Todos los archivos se guardan en su computadora. No se envía información a ningún servidor externo.

Advertencias Importantes

  • Guarde su progreso frecuentemente (cada 15-20 minutos)
  • Si cierra el navegador sin guardar, perderá su trabajo
  • Los archivos JSON son texto plano, puede editarlos manualmente si es necesario
  • Necesita conexión a internet solo para la primera carga de la página

Preguntas Frecuentes

¿Puedo usar esta aplicación sin internet?

Necesita internet solo para cargar la página por primera vez (descarga las librerías). Después puede trabajar offline.

¿Mis datos están seguros?

Sí, todos los datos se procesan localmente en su navegador. No se envía información a ningún servidor.

¿Qué navegador es mejor?

Recomendamos Google Chrome o Microsoft Edge para mejor rendimiento, pero funciona en cualquier navegador moderno.

¿Puedo editar múltiples archivos RVTools?

Puede cargar un archivo a la vez. Para múltiples archivos, procese uno, exporte, y luego cargue el siguiente.

¿Qué hago si encuentro un error?

Presione F12 para abrir la consola del navegador y vea el mensaje de error. Verifique que su archivo RVTools sea válido y contenga la hoja "vInfo".

Mejores Prácticas

  1. Revise primero las validaciones automáticas
  2. Trabaje por grupos (use filtros por cluster o datacenter)
  3. Complete primero los servidores productivos
  4. Guarde progreso cada 15-20 minutos
  5. Verifique las estadísticas para asegurar completitud
  6. Exporte a Excel y PDF al finalizar
